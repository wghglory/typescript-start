装饰器模式（Decorator Pattern）允许向⼀个现有的对象添加新的功能，同时⼜不改变其结构。这种类型的设计模式属于结构型模式，它是作为现有的类的⼀个包装。

这种模式创建了⼀个装饰类，⽤来包装原有的类，并在保持类⽅法签名完整性的前提下，提供了额外的功能。
